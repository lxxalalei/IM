## 0. 执行约束

本项目后续规划、设计、编码和交付必须遵守 [项目工作规则](./WORKING_RULES.md)，尤其是其中的“置信度审查机制”：每个关键策略和实现方案都需要先审查漏洞、修正风险，再继续推进。

## 1. 产品概述

本产品是一款面向团队协作和轻量办公场景的即时通讯客户端，首版聚焦“消息沟通”主链路，采用 Flutter 开发，优先实现桌面端体验，同时保留移动端响应式扩展空间。

产品形态参考飞书桌面端：左侧全局导航栏、中间会话列表、右侧聊天详情区，整体风格轻量、克制、信息密度高，以浅色办公界面为主。

## 2. 产品目标

1. 构建一个可运行的 Flutter IM MVP，具备类飞书的基础交互和视觉骨架。
2. 完成从登录态模拟、会话列表、聊天窗口、消息发送到本地消息展示的闭环。
3. 建立清晰的模块边界，为后续接入真实后端、WebSocket、联系人、群聊、文件、机器人等能力留出结构。
4. 用 PRD 驱动后续 vibecoding，每一阶段都有明确交付物和验收标准。

## 3. 目标用户

### 3.1 核心用户

- 小团队成员：需要日常沟通、项目同步、资料传递。
- 运营/客服/协作岗位：高频使用会话列表、群聊、机器人通知。
- 管理者：需要快速查看组织联系人、团队动态、未读消息。

### 3.2 用户诉求

- 快速找到人或群，并开始沟通。
- 明确知道哪些会话有未读、哪些消息重要。
- 消息输入、发送、表情、附件入口要顺手。
- 界面不能打扰工作，要清爽、稳定、响应快。

## 4. 产品定位

短期定位：团队内部 IM 桌面客户端原型。  
中期定位：支持联系人、群组、机器人、文件消息、搜索的协作 IM。  
长期定位：向轻量办公套件扩展，包含日历、文档、会议、工作台等模块。

## 5. 设计原则

1. 类飞书但不照搬：布局、信息密度、交互节奏参考飞书，颜色、图标、文案可形成自有品牌。
2. 先桌面后移动：首版优先 1200px 以上桌面布局，后续适配平板和手机。
3. 信息优先：视觉装饰克制，让头像、会话、消息、未读状态成为主体。
4. 状态完整：空状态、加载态、错误态、选中态、悬停态、禁用态都要预留。
5. 模块可替换：MVP 可以先用本地 mock 数据，但架构应便于替换为真实 API。

## 6. 范围定义

### 6.1 MVP 必做

- 应用主框架：左侧全局导航栏、中间会话列表、右侧聊天面板。
- 消息模块：单聊会话展示、消息气泡、日期分割、消息输入、发送消息。
- 会话模块：会话列表、选中会话、未读/时间/摘要展示、置顶样式预留。
- 搜索入口：全局搜索 UI 入口，首版可展示静态搜索框或弹窗。
- 用户状态：当前用户头像、在线状态、基础个人信息 mock。
- 输入工具栏：文本输入、表情、@、附件、截图/剪刀、更多入口的 UI。
- 本地 mock 数据：联系人、会话、消息。
- 基础响应式：桌面宽度变化时三栏不破版。

### 6.2 MVP 可选

- 会话筛选：全部、未读、单聊、群聊。
- 机器人会话标签。
- 消息发送中的 pending 状态。
- 简单右键菜单或更多菜单。
- 暗色模式 token 预留。

### 6.3 首版不做

- 真实登录注册。
- WebSocket、推送通知。
- 音视频会议。
- 云文档编辑。
- 复杂组织架构管理。
- 端到端加密。
- 多端消息同步。
- 富文本编辑器完整能力。

## 8. 核心页面需求

### 8.1 应用主框架

#### 页面结构

- 左侧全局导航栏：固定宽度，约 64px。
- 中间会话列表栏：固定或弹性宽度，建议 288px。
- 右侧聊天面板：占据剩余空间，最小宽度建议 520px。

#### 视觉要求

- 背景采用浅蓝灰色应用底色。
- 内容区域使用白色或接近白色。
- 分割线要轻，避免强边框。
- 选中态使用浅蓝色块。
- 圆角克制，普通容器建议 6-8px。

#### 验收标准

- 1200x720 下结构与参考图接近。
- 缩放到 900px 宽时不出现文本严重重叠。
- 当前所在模块有明确选中态。

### 8.2 全局导航栏

#### 功能项

- 顶部个人头像，显示在线状态。
- 新建按钮。
- 主导航：
  - 消息
  - 云文档
  - 多维表格
  - 视频会议
  - 知识库
  - 工作台
  - 通讯录
  - 日历
  - 飞书社/更多，可作为占位

#### 交互

- 点击导航项切换右侧主内容。
- MVP 中非消息模块可显示“建设中”占位页。
- 悬停显示 tooltip。

#### 验收标准

- 图标、文字上下排列。
- 消息模块默认选中。
- 窄栏布局稳定，不随文字长度改变宽度。

### 8.3 会话列表

#### 元素

- 顶部标题：消息。
- 顶部操作：可预留折叠菜单/排序入口。
- 快捷联系人头像区：横向展示 3-5 个常用联系人。
- 会话列表项：
  - 头像
  - 会话名
  - 标签：外部、官方、机器人、智能体等
  - 最后一条消息摘要
  - 时间
  - 未读数
  - 置顶/免打扰状态预留

#### 交互

- 点击会话，右侧切换聊天记录。
- 当前会话高亮。
- 长标题和摘要单行省略。
- 未读会话可使用加粗标题和未读角标。

#### 验收标准

- 会话列表滚动顺畅。
- mock 数据不少于 10 条。
- 至少包含联系人、机器人、系统通知三类会话。

### 8.4 聊天详情

#### 顶部栏

- 左侧：头像、会话名、标签。
- 右侧操作：
  - 搜索
  - 语音/电话
  - 添加成员
  - 日历/任务入口占位
  - 更多

#### 消息区

- 日期分割线。
- 对方消息：头像在左，气泡浅灰。
- 我方消息：头像或气泡靠右，气泡可使用浅蓝。
- 系统消息：居中淡色文本。
- 支持滚动到底部。

#### 输入区

- 单行或多行输入框。
- placeholder：发送给 {会话名}。
- 工具栏图标：
  - 字体/格式
  - 表情
  - @
  - 截图
  - 附件
  - 展开
  - 发送
- Enter 发送，Shift+Enter 换行。

#### 验收标准

- 切换会话时标题、消息、placeholder 同步变化。
- 输入文本后点击发送或按 Enter 可追加到消息列表。
- 空输入不可发送，发送按钮显示禁用态。

### 8.5 搜索

#### MVP 表现

- 应用顶部固定全局搜索框。
- 搜索框后续可打开搜索弹窗，包含最近搜索/快捷项占位。
- 搜索结果可暂时基于 mock 会话名过滤。

#### 后续扩展

- 搜索联系人。
- 搜索聊天记录。
- 搜索文件。
- 搜索云文档。

## 9. 核心流程

### 9.1 打开应用

1. 应用启动。
2. 进入 mock 登录态。
3. 默认打开“消息”模块。
4. 默认选中第一条会话。
5. 右侧展示该会话聊天记录。

### 9.2 发送消息

1. 用户选择一个会话。
2. 在底部输入框输入文本。
3. 点击发送或按 Enter。
4. 消息追加到当前会话底部。
5. 会话列表摘要更新为最新消息。
6. 会话时间更新为当前时间。

### 9.3 切换会话

1. 用户点击会话列表项。
2. 会话列表选中态切换。
3. 右侧顶部标题更新。
4. 消息区展示对应历史消息。
5. 该会话未读数清零。

## 11. Flutter 技术规划

### 11.1 推荐技术栈

- Flutter stable。
- 状态管理：Riverpod 或 Provider，MVP 推荐 Riverpod。
- 路由：go_router，MVP 可先用单页 Shell。
- 本地数据：内存 mock，后续可接 Drift/Hive。
- 资源管理：本地 SVG/icon font 或 lucide 风格图标库。
- 网络层：Dio 预留。
- 实时通信：WebSocket 预留。

### 11.3 状态拆分

- `currentNavProvider`：当前全局导航。
- `conversationListProvider`：会话列表。
- `selectedConversationIdProvider`：当前选中会话。
- `messagesProvider(conversationId)`：某个会话的消息。
- `messageDraftProvider(conversationId)`：输入草稿。

## 12. 后端技术规划

### 12.1 MVP 技术栈

- 后端框架：FastAPI。
- 数据库：SQLite，本地文件路径 `backend/data/vibe_im.sqlite3`。
- 数据访问：MVP 使用 Python 标准库 `sqlite3`，降低依赖和迁移成本。
- API 风格：REST JSON。
- 跨域：允许本地 Flutter Web 预览地址访问。
- 实时通信：后续版本增加 WebSocket，不阻塞 REST MVP。

### 12.2 后端目录

```text
backend/
├── app/
│   ├── __init__.py
│   ├── main.py
│   └── database.py
├── data/
│   └── vibe_im.sqlite3
├── tests/
├── requirements.txt
└── README.md
```

### 12.3 MVP API

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/health` | 健康检查 |
| GET | `/api/conversations` | 获取会话列表 |
| GET | `/api/conversations/{id}` | 获取单个会话 |
| GET | `/api/conversations/{id}/messages` | 获取会话消息 |
| POST | `/api/conversations/{id}/messages` | 发送文本消息 |
| PATCH | `/api/conversations/{id}/read` | 清除会话未读 |
| GET | `/api/contacts` | 获取联系人列表，支持 `q` 搜索 |
| GET | `/api/contacts/{id}` | 获取联系人详情 |

## 15. MVP 迭代计划

### Milestone 1：项目初始化和视觉骨架

目标：创建 Flutter 项目，完成三栏主界面静态布局。

交付：
- Flutter 项目可运行。
- 全局导航栏、会话列表、聊天面板完成静态 UI。
- 主题 token 初步建立。

验收：
- 桌面窗口打开后第一屏接近参考图布局。
- 不依赖后端。

### Milestone 2：Mock 数据和会话切换

目标：让会话列表和聊天面板由 mock 数据驱动。

交付：
- User/Conversation/Message 模型。
- mock 会话和消息数据。
- 点击会话切换右侧聊天内容。

验收：
- 至少 10 个会话。
- 至少 3 个会话有不同聊天记录。
- 选中态和未读清除逻辑可用。

### Milestone 3：消息输入和发送

目标：完成本地发送消息闭环。

交付：
- 输入框。
- Enter/点击发送。
- 消息追加。
- 会话摘要和时间更新。

验收：
- 空消息不能发送。
- 发送后输入框清空。
- 新消息自动滚动到可见区域。

### Milestone 4：搜索和基础弹窗

目标：实现基础搜索体验。

交付：
- 全局搜索弹窗。
- 按会话名过滤 mock 会话。
- 点击结果跳转对应会话。

验收：
- 搜索框可输入。
- 搜索结果实时变化。
- 选择结果后关闭弹窗并进入会话。

### Milestone 4.5：本地后端和数据库

目标：搭建本地后端，让 IM MVP 从 mock 数据逐步迁移到 API 数据。

交付：
- FastAPI 应用。
- SQLite 本地数据库。
- 会话和消息表。
- 会话列表、消息列表、发送消息、清除未读 API。

验收：
- `GET /health` 返回正常。
- 数据库文件自动创建在本机。
- 可通过 API 获取 seed 会话和消息。
- 可通过 API 写入新消息并更新会话摘要。

### Milestone 5：体验补齐

目标：完善视觉细节和状态。

交付：
- hover、selected、disabled 状态。
- 空状态、建设中页面。
- 基础响应式。
- 简单动画。

验收：
- 900-1440px 宽度主要布局正常。
- 图标按钮有 tooltip。
- 文本不明显溢出。

## 17. 风险和决策点

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 过早接入复杂后端 | 拖慢 MVP | 后端先使用 FastAPI + SQLite，仅覆盖 IM 主链路 |
| 视觉过度照搬飞书 | 品牌和版权风险 | 借鉴布局和体验，形成自有颜色、图标和命名 |
| Flutter 桌面细节不足 | 体验粗糙 | 优先打磨 hover、滚动、输入框、窗口尺寸 |
| 状态管理混乱 | 后续难扩展 | 从 MVP 开始按 feature 分层 |
| 消息模型过窄 | 后续重构成本高 | 首版预留 message type 和 status |

## 18. 验收总标准

MVP 完成时，应满足：

1. 可以运行 Flutter 桌面应用。
2. 第一屏视觉结构明显接近参考截图。
3. 默认进入消息页，并选中一个会话。
4. 用户可以切换会话。
5. 用户可以发送本地文本消息。
6. 会话列表会更新最新消息摘要。
7. 搜索入口可用，至少能搜索 mock 会话。
8. 本地后端可启动，SQLite 数据库可自动创建。
9. 非 MVP 模块有统一占位页。
10. 核心组件已拆分，为后续接后端留出空间。
